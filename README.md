# LunchBot — бот для заказа обедов

Telegram-бот для офиса: опрос по меню, приём оплаты PDF-чеками, каталог блюд
с фото, расчёт для админа.

Бот `@tunduk_obed_bot`. Работает локально на Windows через IntelliJ IDEA.

---

## Запуск на Windows

### 1. JDK 21

Скачать Temurin 21 (https://adoptium.net). В установщике отметить
**Set JAVA_HOME** и **Add to PATH**. Проверка в новом терминале:

```
java -version
```

### 2. Postgres

Поставить PostgreSQL (https://www.postgresql.org/download/windows/), pgAdmin идёт в комплекте.
Пароль пользователя `postgres` задать `postgres`.

В pgAdmin создать базу `lunch_bot_db`, затем в Query Tool на ней:

```sql
CREATE EXTENSION IF NOT EXISTS fuzzystrmatch;
```

### 3. Открыть проект в IDEA

`File - Open` - папку с `pom.xml`. Дождаться загрузки Maven.

`Settings - Build - Compiler - Annotation Processors` - включить
**Enable annotation processing** (нужно для Lombok).

`File - Project Structure - Project` - SDK **21**.

### 4. Переменные окружения

`Run - Edit Configurations - + - Spring Boot`, main class
`com.example.lunchbot.LunchBotApplication`, в поле **Environment variables**:

```
TELEGRAM_BOT_TOKEN=7986449169:AAEOAXLT6N_1Hh806gywPClZEH37DwJ-PJY
LUNCH_ADMIN_CHAT_ID=ваш_id_из_@userinfobot
```

Если пароль Postgres не `postgres`, добавить `SPRING_DATASOURCE_PASSWORD`.

### 5. Запуск

`Shift+F10`. В логе должно появиться:

```
Successfully applied 10 migrations to schema "public"
Telegram long polling запущен
Started LunchBotApplication
```

### 6. Настройка в Telegram

В @BotFather: `/setprivacy` - выбрать бота - **Disable**.
Без этого бот не видит команды в группе.

Добавить бота в группу, сделать **администратором**.

В основной группе выполнить:

```
/setmain
/recipient
```

## Ошибка миграций (checksum mismatch)

Если Flyway пишет «Migrations have failed validation» — база помнит старую
версию миграции. Пока данных не жалко, остановить бота и в pgAdmin:

```sql
DROP SCHEMA public CASCADE;
CREATE SCHEMA public;
CREATE EXTENSION IF NOT EXISTS fuzzystrmatch;
```

Запустить бота — миграции накатятся заново.

## Как работает оплата

### Цена

- столовая берёт **240** за порцию
- человек платит **250**, кроме льготного списка — те платят **240**
- список льготников в переменной `LUNCH_PRICE_REDUCED_USERS`
- разница (10 сом) — маржа админа

### Чек

Только **PDF** — в нём есть текстовый слой с суммой. Фото и скриншоты
отклоняются. Бот проверяет: получателя (телефон или имя), дату и сумму.

Оплата действует в пределах одного опроса. Прислал 500 — можно взять два
блюда. Один и тот же чек дважды не пройдёт (защита по номеру транзакции).

Чек на **старый** номер получателя отклоняется с пояснением, что деньги
ушли не туда.

### ELQR и QR-оплата

В чеках, оплаченных по QR (ELQR), номер получателя не печатается — он зашит
в код. Такие чеки засчитываются по имени получателя и дате.

### Бесплатное блюдо

Получатель платежа (тот, чей id задан в `/recipient`) берёт **первое** блюдо
без оплаты. Следующие — как всем, за чек.

### Несколько порций

Чтобы взять вторую порцию (в том числе того же блюда) — нажать
«➕ Добавить ещё блюдо», потом выбрать блюдо. В опросе покажется «Азиз ×2».

## Команды

### Для всех

| Команда | Где | Что делает |
|---|---|---|
| `/start` | личка | приветствие |
| `/menu` | группа | фото блюд текущего опроса |
| `/qr` | группа/личка | показать QR для оплаты |

Оплата: под опросом кнопка «Подтвердить оплату» ведёт в личку бота,
там присылаете PDF-чек.

### Для админа

| Команда | Где | Что делает |
|---|---|---|
| `/lunch` + меню | группа | создать опрос по блюдам |
| `/close` | группа | закрыть опрос |
| `/summary` | группа | итог для столовой (подпись Батырхан) |
| `/money` | группа/личка | денежный расчёт, ответ приходит в личку |
| `/catalog` | группа | каталог постоянных блюд |
| `/dish <название>` + фото | группа | привязать фото к блюду |
| `/recipient` | группа | сменить реквизит получателя (пошагово) |
| `/setqr` + картинка | группа | сохранить QR для оплаты |
| `/setmain` | группа | отметить основную группу |
| `/cancel` | группа | выйти из текущего диалога |

### Создание опроса

```
/lunch
Здравствуйте

Босо лагман
Гюро лагман
Бифштекс с гарниром
```

Приветствие и строки из одних эмодзи отбрасываются. Каждое блюдо — с новой строки.

### Кнопки под опросом

- **блюдо** — выбрать (нужна оплата)
- **🧾 Подтвердить оплату** — открывает личку бота для отправки чека
- **➕ Добавить ещё блюдо** — разрешает второй выбор
- **❌ Отменить мои выборы** — снимает все выборы разом

### /money

Приходит только админу в личку:

```
Порций: 17 оплаченных + 1 бесплатное
Столовой: 17 × 240 = 4080
Собрано: 4470
Остаток вам: 390

Переплатили:
  Азамат — 240 (взял 1, оплатил 490)

Оплатили, но не выбрали блюдо:
  Марат — 250 (вернуть полностью)
```

## Списки команд для BotFather

@BotFather - `/setcommands` - выбрать бота - вставить список.

BotFather показывает **один** список всем. Рекомендуется залить короткий:
админские команды всё равно работают, просто не маячат у всех.

### Для пользователей (рекомендуется)

```
start - начать
menu - фото сегодняшних блюд
qr - QR для оплаты
```

### Полный список

```
start - начать
menu - фото сегодняшних блюд
qr - QR для оплаты
lunch - создать опрос по меню
close - закрыть опрос
summary - итог для столовой
money - денежный расчёт (в личку)
catalog - каталог блюд
recipient - сменить реквизит получателя
setqr - сохранить QR (с картинкой)
setmain - отметить основную группу
cancel - отменить текущее действие
```

Команды с аргументами (`/dish Название`) в BotFather не добавляются —
они работают, но в подсказке при вводе `/` не появятся.

## Переменные окружения

| Переменная | Значение |
|---|---|
| `TELEGRAM_BOT_TOKEN` | токен от @BotFather |
| `LUNCH_ADMIN_CHAT_ID` | ваш user_id (@userinfobot), туда приходят чеки на ручную проверку |
| `LUNCH_RECEIPT_BACK_DAYS` | 0 — только сегодняшний чек, 1 — принимать и вчерашний |
| `LUNCH_PRICE_REDUCED_USERS` | id тех, кто платит 240, через запятую |

Реквизит получателя и основная группа хранятся в базе (команды `/recipient`,
`/setmain`), в переменных не задаются.

## Полезные запросы к базе

Кто голосовал (id, имя, ник):

```sql
SELECT DISTINCT user_id, full_name, username FROM lunch_poll_vote ORDER BY full_name;
```

Текущие настройки реквизита:

```sql
SELECT key, value FROM app_setting;
```

Каталог блюд:

```sql
SELECT id, name, active, photo_file_id IS NOT NULL AS has_photo FROM dish;
```

## Что помнить

- **Не запускать два экземпляра** с одним токеном — они воруют апдейты друг
  у друга, бот отвечает через раз.
- **Миграции не редактировать** после применения — только добавлять новые `V11+`.
  Иначе Flyway откажется стартовать.
- После замены java-файлов делать **Build - Rebuild Project**, иначе запустится
  старая скомпилированная версия.
- Бэкап базы: `pg_dump -U postgres lunch_bot_db > backup.sql`
